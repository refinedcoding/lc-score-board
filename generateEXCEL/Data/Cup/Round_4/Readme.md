记f(n, a, b)、g(n, a, b)分别为最早和最晚。不失一般性，设a < b, a + b <= n + 1。
从而大致的位置只有两种可能：  a b | ..... 与  a    | b

1. 最早：
    ①. b为中点或者在中点右侧，设a左边x个人，s人赢，b右边y(>=x)个人，y-a人中t人赢，剩余a人应为x-s人赢，
注意到t的取值范围: 0 <= t <= y - a。
        (i). x = y，也即a-1=n-b, a + b = n + 1, 此时f(n, a, b) = 1
        (ii). x != y, 一轮后a左边剩s人，b右边剩x-s+t人，贪心联立有2s=x+t，故若y-a>0, 也即a + b < n, 我们取t = x % 2, 则有f(n,a,b) = 2
	        若y = a, 也即 a + b = n，我们想尝试挑一个尽量小的|x-2s| -> s = [x/2], 或者干脆|x-2s|>1以保证我们下一次能达到a + b < n。
                        对于后一种情况，考虑到操作后a与b会跑到严格同一侧，故|x-2s|也不宜过大，从而我们选择s = [(x-2)/2]。对应起来就是
		f(n, a, b) = 1 + min(f(m, (a + 1) / 2, m - a / 2), f(m, (a - 1) / 2, m - 1 - a / 2)), 这里 m = [(n+1)/2]
    ②. b与a严格在中点左侧。同令m = [(n+1)/2]。这一轮，显然f(n, a, b)>1，令a和b之间人数分布为x1y1其中两个1依次表示a和b，
         令s, v表示x, y中获胜人数，则下一轮开场，a左侧有s人，b左侧有s + v + 1人，所以b右侧有m-s-v-2人。考虑2s+v+2与m的关系,
         显然s<=a-1, v <= b - a - 1, 2s+v+2 <= a + b - 1：
         (i). 若a + b - 1 = m, 取s=a-1, v = b-a-1, 有f(n, a, b) = 2;
         (ii). 若a + b - 1 > m, 则考虑v的取值: 若b-a>1, 则令v= m%2, s = (m-2-v)/2; 若b-a=1, 如果2|m，那么仍取v= m%2, s = (m-2-v)/2即可；
              这些情况均有f(n, a, b) = 2。
              如果2不整除m，此时，a和b是邻居，那让a和b尽量靠近中线就行，取s = [(m-2)/2]=(m-3)/2, f(n, a, b) = 1 + f(m, s + 1, s + 2).
         (iii). 若a + b < m, 贪心直接让b左侧的人全胜，这样若下一轮b在中线右侧，则为①(ii)的情况，若b不在中线右侧也不亏~f(n, a, b) = 1 + f(m, a, b).
         (iv). 若a + b = m, 仍对应①(ii)的情况，由讨论知除了让b左侧的人全胜，以尽快让b到达中点右侧外，我们同时考虑牺牲一点“速度”而让之后的a'+b'<m，
	故f(n, a, b) = 1 + min(f(m, a, b), f(m, a-1, b-1));
  
2. 最晚：
    ①. b为中点或者在中点右侧，若a + b = n + 1，则g(n, a, b) = 1; 若a + b < n + 1, 我们直接删除a左侧所有的人以及尽量删除b左侧的人，得到
         g(n, a, b) = 1 + g(m, 1, b + 1 - n / 2)
    ②. a与b同侧，直接删至a, b相邻且在最左侧： g(n, a, b) = 1 + g(m, 1, 2)
